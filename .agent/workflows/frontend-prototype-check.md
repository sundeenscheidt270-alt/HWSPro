# 前端原型质量检查清单

**使用场景**: AI生成前端原型后，按此流程逐项检查质量，确保符合需求和标准。

---

## 第一部分：PRD符合性检查

### 1.1 字段一致性
- [ ] Mock数据字段是否与PRD的字段字典一致？
- [ ] 字段类型是否正确（字符串/数字/日期）？
- [ ] 必填字段在表单中是否标记为必填？

### 1.2 枚举一致性
- [ ] 状态枚举值是否使用PRD定义的值？
- [ ] 状态文案是否与PRD一致？
- [ ] 下拉选项是否与PRD一致？

### 1.3 操作一致性
- [ ] 页面级操作按钮是否与PRD一致？
- [ ] 行级操作按钮是否与PRD一致？
- [ ] 批量操作是否与PRD一致？

### 1.4 流程一致性
- [ ] 状态流转是否遵循PRD的状态机定义？
- [ ] 操作按钮在不同状态下的显示/隐藏是否正确？

---

## 第二部分：功能完整性检查

### 2.1 查询与筛选
- [ ] 查询条件是否完整？
- [ ] 关键词搜索是否工作？
- [ ] 状态筛选下拉是否工作？
- [ ] 日期范围筛选是否工作？
- [ ] 清空筛选条件是否工作？

### 2.2 列表展示
- [ ] 列表字段是否完整？
- [ ] 数据展示是否正确？
- [ ] 排序是否工作？
- [ ] 分页是否工作？
- [ ] 每页条数是否合理（10-20条）？

### 2.3 表单操作
- [ ] 新增表单能否打开？
- [ ] 必填项校验是否生效？
- [ ] 保存后列表是否更新？
- [ ] 编辑表单能否打开？
- [ ] 编辑表单是否回填数据？
- [ ] 编辑保存后列表是否更新？

### 2.4 其他操作
- [ ] 删除操作是否有二次确认？
- [ ] 删除后列表是否更新？
- [ ] 批量操作是否工作？
- [ ] 详情查看是否工作？

---

## 第三部分：交互体验检查

### 3.1 弹窗/抽屉
- [ ] 弹窗能否正常打开？
- [ ] 弹窗能否正常关闭？
- [ ] Esc键能否关闭弹窗？
- [ ] 点击外部能否关闭弹窗？
- [ ] 弹窗打开时是否聚焦到第一个输入框？

### 3.2 表单交互
- [ ] 必填项未填写时是否有红色提示？
- [ ] 格式错误时是否有错误提示？
- [ ] Enter键能否提交表单？
- [ ] 表单重置是否工作？

### 3.3 反馈机制
- [ ] 操作成功时是否有Toast提示？
- [ ] 操作失败时是否有Toast提示？
- [ ] Toast是否3秒后自动消失？
- [ ] 按钮点击时是否有加载状态？

### 3.4 空态/错误态
- [ ] 是否有"演示开关"可切换状态？
- [ ] 加载态是否实现（Loading）？
- [ ] 空数据态是否实现（Empty）？
- [ ] 错误态是否实现（Error）？
- [ ] 空态/错误态是否有引导文案？

---

## 第四部分：可访问性检查

### 4.1 键盘导航
- [ ] Tab键能否正常导航？
- [ ] Tab键是否会穿透遮罩？（不应该）
- [ ] Esc键能否关闭弹窗/下拉？
- [ ] Enter键能否提交表单？

### 4.2 焦点管理
- [ ] 弹窗打开时是否自动聚焦？
- [ ] 焦点是否在第一个可交互元素上？
- [ ] 弹窗关闭后焦点是否回到触发元素？

### 4.3 其他可访问性
- [ ] 按钮是否有合适的aria-label？
- [ ] 表单字段是否有label？
- [ ] 错误提示是否与字段关联？

---

## 第五部分：视觉质量检查

### 5.1 布局
- [ ] 顶部导航是否完整？
- [ ] 左侧菜单是否可折叠？
- [ ] 主内容区布局是否合理？
- [ ] 指标卡是否显示（≥3个）？

### 5.2 样式
- [ ] 整体配色是否协调？
- [ ] 间距是否统一？
- [ ] 圆角是否统一？
- [ ] 阴影是否合理？
- [ ] 字体大小是否合适？

### 5.3 响应式（可选）
- [ ] 窗口缩小时布局是否正常？
- [ ] 表格是否有横向滚动？

---

## 第六部分：数据质量检查

### 6.1 Mock数据
- [ ] Mock数据是否充分（≥20条）？
- [ ] 数据是否符合业务逻辑？
- [ ] 是否包含各种状态的数据？
- [ ] 金额格式是否正确（千分位、2位小数）？
- [ ] 日期格式是否统一？

### 6.2 数据展示
- [ ] 枚举值是否显示文案（而非数字）？
- [ ] 金额是否显示千分位？
- [ ] 日期是否格式化显示？

---

## 第七部分：代码质量检查

### 7.1 代码组织
- [ ] 是否有统一的state对象？
- [ ] 是否有render()渲染函数？
- [ ] 事件是否使用事件委托？
- [ ] 是否有关键逻辑的注释？

### 7.2 命名规范
- [ ] 变量命名是否语义化？
- [ ] 函数命名是否以动词开头？
- [ ] DOM ID命名是否使用短横线？

### 7.3 安全性
- [ ] 用户输入是否做了转义？
- [ ] 表单是否有基本校验？

---

## 第八部分：性能检查

- [ ] 页面加载是否流畅（无明显卡顿）？
- [ ] 列表渲染是否流畅？
- [ ] 筛选操作是否即时响应？
- [ ] 是否有不必要的全量渲染？

---

## 检查完成后的行动

### 如果发现问题（重要）
根据问题类型使用不同Prompt：

**PRD符合性问题**：
```
@生成的index.html
@PRD文档

我检查发现以下PRD符合性问题：
1. [具体问题]
2. [具体问题]

请根据PRD修正
```

**功能或交互问题**：
```
@生成的index.html

我检查发现以下功能问题：
1. [具体问题]
2. [具体问题]

请修复这些问题
```

**代码质量问题**：
```
@生成的index.html
@frontend-prototype-guide.md

我检查发现以下代码质量问题：
1. [具体问题]
2. [具体问题]

请按照规范优化
```

### 如果通过检查
- 保存HTML文件
- 准备演示材料
- 安排需求评审

---

## 常见问题提醒

⚠️ **经常被遗漏的10个点**：
1. Esc键关闭弹窗（最常遗漏）
2. 点击外部关闭弹窗
3. 删除操作的二次确认
4. Toast自动消失
5. 弹窗打开时的焦点管理
6. 空数据态提示
7. 状态流转的按钮显示/隐藏逻辑
8. 金额的千分位格式化
9. 枚举值显示文案而非数字
10. 演示开关（切换加载/空数据/错误态）
