## 1. 项目概述

### 1.1 背景与现状(Why)

**现状描述**:

- 维他很忙拥有 5000+SKU，1000 家门店，每日采购订单量 300-500 单
- 当前采购流程手工化程度高，从需求汇总到下单平均需要 2-3 天
- 多系统割裂：采购计划在 Excel、订单在邮件、收货在纸质单据
- 数据不及时：供应商交期延迟无法实时预警，导致门店断货

**关键痛点**:

1. **采购效率低**：手工汇总门店需求，每天耗时 4 小时+，人工错误率高达 5%
2. **订单追踪难**：无法实时查看订单执行状态，供应商延期无预警
3. **收货混乱**：纸质单据易丢失，收货差异处理耗时长，平均 3-5 天
4. **成本难控**：无法实时掌握采购成本，缺少历史价格对比
5. **数据孤岛**：采购数据无法与库存、财务系统实时同步

**影响**:

- **效率损失**：采购员每天 4 小时重复劳动，人效低下
- **成本损失**：无价格监控，采购成本难控，年损失约 500 万
- **库存损失**：因订单延期导致门店断货，月均销售损失 200 万
- **体验损失**：断货影响客户体验，会员流失率上升

### 1.2 目标与价值(What / Value)

**业务目标(可量化)**:

- 采购订单处理时效从 2-3 天缩短至 4 小时内(提升 80%+)
- 采购数据准确率从 95%提升至 99.9%(错误率降低 98%)
- 订单追踪及时率达 100%(当前 0%)
- 供应商准时交货率从 75%提升至 90%
- 采购成本透明度达 100%(当前 30%)

**用户目标**:

- **采购员**：自动汇总需求，快速生成采购订单，减少重复劳动
- **采购主管**：实时掌握订单执行进度，供应商交期预警
- **仓管员**：便捷收货流程，快速处理收货差异
- **财务**：自动生成应付账款，成本数据实时准确

**系统目标**:

- 建立端到端采购流程闭环(需求 → 订单 → 收货 → 对账)
- 实现采购全流程数字化、可追溯
- 与 WMS/SRM/财务系统集成，数据实时同步
- 支持日均 500 单处理能力，峰值 1000 单

### 1.3 成功指标与口径

### 1.4 范围边界(Scope)

**In Scope**

- 采购订单创建(手动/导入/API 生成)
- 采购订单管理(查询/修改/删除)
- 采购订单审批(多级审批流程)
- 订单发送供应商(通过 SRM 系统)
- 收货通知接收(ASN)与展示
- 收货单管理(与 WMS 集成)

- 采购报表(订单统计/供应商绩效/成本分析)

- 采购付款(财务系统负责)
- 采购预测与智能推荐(Phase 2 考虑)

**约束条件**

- **时间约束**：Phase 1 需在 3 个月内上线(2026-04-01 前)
- **技术约束**：必须与 SRM/WMS/财务系统集成
- **合规约束**：符合公司财务审计要求，关键操作留痕 6 个月
- **资源约束**：开发团队 3 人，测试 1 人
- **历史数据**：需迁移近 1 年采购历史数据(约 10 万单)

## 2. 业务概览(面向"共识")

### 2.1 业务参与方与组织模型

**关键角色**:

- **采购计划员**：汇总需求，创建采购订单
- **采购主管**：审批采购订单(\u003c10 万)
- **采购总监**：审批大额采购订单(≥10 万)
- **供应商**：接收 PO，发货
- **仓管员**：收货验货，录入收货单
- **质检员**：质检商品，判定合格/不合格
- **财务**：生成应付账款，对账结算

**涉及系统**:

- **ERP(本系统)**：采购订单管理核心
- **SRM(供应商关系管理)**：供应商档案、PO 推送、ASN 接收
- **WMS(仓库管理)**：收货作业、库存更新
- **财务系统**：应付账款、成本核算
- **门店补货系统**：生成补货需求
- **PMS(商品管理)**：商品主数据来源

详细集成见 [6.系统功能清单和上下游集成系统.md](../../docs/6.系统功能清单和上下游集成系统.md)

### 2.2 端到端业务流程概览

#### 主流程：采购到入库

```mermaid
graph TD
    A[1.採购需求\u003cbr/\u003e门店补货/库存补货/促销] --\u003e B[2.创建采购订单\u003cbr/\u003ePO]
    B --\u003e C{3.PO审批}
    C --\u003e|驳回| B
    C --\u003e|通过| D[4.发送PO给供应商\u003cbr/\u003e通过SRM]

    D --\u003e E[5.供应商确认PO\u003cbr/\u003e在SRM系统]
    E --\u003e F[6.供应商发货\u003cbr/\u003e生成ASN]

    F --\u003e G[7.ASN推送WMS\u003cbr/\u003e仓库准备收货]
    G --\u003e H[8.供应商送货到仓]

    H --\u003e I[9.仓库收货\u003cbr/\u003e数量清点]
    I --\u003e J{10.质检IQC}

    J --\u003e|不合格| K[拒收/退货\u003cbr/\u003e生成差异单]
    J --\u003e|合格| L[11.上架入库\u003cbr/\u003eWMS操作]

    L --\u003e M[12.回传入库确认\u003cbr/\u003eWMS→ERP]
    M --\u003e N[13.ERP处理\u003cbr/\u003e更新PO状态+增加库存+生成应付]

    K --\u003e O[供应商处理\u003cbr/\u003e补发或索赔]
    O --\u003e F
```

**关键节点**:

1. 采购需求产生(门店补货/库存低于补货点/促销活动)
2. 创建采购订单(手动/批量导入)
3. 采购订单审批(多级审批根据金额)
4. 发送 PO 给供应商(通过 SRM 推送)
5. 供应商确认 PO
6. 供应商发货并生成 ASN
7. ASN 推送到 WMS
8. 仓库收货 验货
9. 质检(IQC)
10. 上架入库
11. 回传入库确认给 ERP
12. ERP 更新库存、PO 状态、生成应付

**关键异常分支**:

- PO 审批驳回 → 采购员修改后重新提交
- 供应商延期发货 → 系统预警，采购主管跟进
- 收货数量短少 → 生成短少单，供应商补发
- 质检不合格 → 拒收，供应商重新发货或退款
- 供应商取消订单 → PO 状态变更为已取消

**同步/异步**:

- ERP→SRM：同步 API 调用，实时推送 PO
- SRM→ERP：异步回调，ASN/确认状态推送
- ERP→WMS：同步 API 调用
- WMS→ERP：异步回调，收货确认推送
- ERP→ 财务：异步消息队列，应付账款生成

**一致性策略**:

- 采购订单：强一致性，ERP 为权威源
- 库存数据：最终一致性，WMS 为权威源
- 应付账款：强一致性，财务系统为权威源

**补偿机制**:

- SRM/WMS 推送失败自动重试(3 次)
- 数据对账(T+1 全量对账)
- 人工补录通道

### 3.2 核心用例(User Story)

#### 用例 1：采购员创建采购订单

**作为**采购计划员  
**我想**快速创建采购订单  
**以便**及时向供应商下单，保证库存

**前置条件**：已登录系统，有采购订单创建权限

**主流程**:

1. 点击"新建采购订单"
2. 选择供应商(下拉搜索)
3. 选择收货仓库
4. 填写期望到货日期
5. 添加商品行：
   - 选择商品(支持条码/名称搜索)
   - 输入采购数量
   - 系统自动带出采购单价(来自供应商商品主数据)
   - 系统自动计算含税金额
6. 可继续添加多行商品
7. 系统自动汇总订单总金额
8. 点击"保存草稿"或"提交审批"

**后置条件**：订单已创建，状态为草稿或待审批

#### 用例 2：采购主管审批采购订单

**作为**采购主管  
**我想**快速审批采购订单  
**以便**控制采购风险，及时下单

**前置条件**：订单状态为"待审批"，有审批权限

**主流程**:

1. 进入"待办任务-采购审批"列表
2. 点击某订单查看详情
3. 查看订单信息：
   - 供应商、金额、商品明细
   - 采购员备注
   - 历史采购价格对比
4. 判断是否通过
5. 填写审批意见
6. 点击"通过"或"驳回"

**后置条件**：

- 通过：订单状态变为"已审批"，自动推送给 SRM
- 驳回：订单状态变为"草稿"，采购员可修改后重新提交

#### 用例 3：仓管员查看收货进度

**作为**仓管员  
**我想**查看预计到货的订单  
**以便**提前准备收货资源

**前置条件**：已登录系统，有收货查询权限

**主流程**:

1. 进入"收货管理-待收货订单"
2. 查看待收货订单列表(按预计到货日期排序)
3. 点击某订单查看详情
4. 查看 ASN 信息(供应商已发货)
5. 查看发货明细(商品、数量、批次号)
6. 准备收货(预留库位、安排人员)

**后置条件**：仓管员知晓到货计划

### 3.3 核心流程(详细)

详细流程已在 2.2 节展示，更详细的流程图见图稿包。

### 3.4 核心数据对象(概览)

**主要对象**:

- **采购订单头(PurchaseOrder)**：订单主信息，包含供应商、总金额、状态等
- **采购订单行(PurchaseOrderLine)**：订单商品明细，包含商品、数量、单价等
- **收货通知(GoodsReceipt)**：仓库收货记录，包含实收数量、收货时间等
- **收货差异(ReceiptVariance)**：收货差异记录，包含差异类型、数量、处理状态等
- **采购历史(PurchaseHistory)**：历史采购价格记录，用于价格分析

详细数据模型见 [附件 A:数据模型与数据字典](./附件A_采购订单数据模型.md)

### 3.5 核心规则(概览)

**关键业务规则**:

1. **PO 编号规则**：PO+YYYYMMDD+4 位序号，如 PO20251228-0001
2. **审批规则**：金额\u003c5 万自动通过，5-10 万采购主管审批，≥10 万采购总监审批
3. **收货差异容差**：实收数量在订购数量 ±2%内视为正常
4. **质检规则**：保质期剩余天数 ≥ 保质期 70%为合格
5. **订单关闭规则**：收货数量 ≥ 订购数量 ×98%可关闭，否则需走差异处理流程
6. **价格波动预警**：采购单价较历史价格上涨\u003e10%需预警
7. **延期预警**：期望到货日期-当前日期\u003c3 天且未收到 ASN 需预警

所有规则详见 [附件 E:规则总表](./附件E_采购订单规则总表.md)

## 4. 非功能需求(NFR)

### 4.1 性能需求

- **响应时间**：订单列表查询\u003c2 秒，订单详情\u003c1 秒，订单创建\u003c3 秒
- **并发**：支持 100 个用户同时操作，峰值 QPS 500
- **数据量**：支持查询近 3 年采购历史(约 50 万单)
- **导入导出**：支持单次导入 500 行，导出 1 万行

### 4.2 可用性需求

- **系统可用性**：≥99.5%(除计划维护外)
- **数据准确性**：订单数据一致性 ≥99.9%
- **API 成功率**：与 SRM/WMS 集成 API 成功率 ≥99%

### 4.3 安全性需求

- **权限控制**：按角色控制功能权限，按组织控制数据范围
- **数据脱敏**：日志中供应商价格脱敏显示
- **操作审计**：关键操作(创建/审批/取消)留痕，保留 6 个月

### 4.4 可维护性需求

- **配置化**：审批金额阈值、收货容差比例可配置
- **监控告警**：订单审批超时、供应商延期实时告警
- **日志审计**：保留 6 个月操作日志，支持查询和分析

### 4.5 用户体验需求

- **便捷性**：支持商品快速搜索(条码/名称)、历史订单复制
- **智能性**：自动带出采购单价、自动计算金额
- **可视化**：订单执行进度可视化、收货进度一目了然
