## 📋 文档清单

### 核心文档

#### 1. [PRD 主档(PRD_Main.md)](./PRD_Main.md) ⭐ 必读

**用途**: 项目概述、业务流程、方案概述、迭代计划

**核心内容**:

- **背景与目标**: 5 大痛点、采购效率提升 80%+、成本透明度 100%
- **业务概览**: 端到端采购流程(需求 → 订单 → 收货 → 对账)
- **功能结构**: 7 大功能模块
  - 采购订单管理(创建/查询/编辑/审批)
  - 供应商协同(PO 发送/ASN 接收/延期预警)
  - 收货管理(收货通知/收货单/进度追踪)
  - 差异处理(短少/多收/质量问题)
  - 订单关闭(自动/手工/强制)
  - 报表与分析(订单统计/供应商绩效/价格历史)
- **集成系统**: SRM、WMS、财务系统
- **迭代计划**: 3 个 Phase、6 个月时间表

**适合**: 所有角色先读本文档建立全局认知

## 📊 项目概览

### 核心目标

- 采购订单处理时效：48 小时 → 4 小时(提升 92%)
- 数据准确率：95% → 99.9%
- 订单追踪覆盖率：0% → 100%
- 供应商准时交货率：75% → 90%

### 功能范围

✅ In Scope:

- 采购订单全生命周期管理
- 多级审批流程
- 与 SRM/WMS/财务集成
- 收货差异处理
- 采购报表与分析

❌ Out of Scope:

- 采购需求计划生成(Phase 2)
- 供应商管理(SRM 系统)
- 采购合同管理(SRM 系统)
- 采购付款(财务系统)

### 迭代计划

- **Phase 1 (3 个月)**: 核心采购订单功能+SRM/WMS 集成
- **Phase 2 (2 个月)**: 差异处理+多级审批
- **Phase 3 (1 个月)**: 智能化+用户体验优化
