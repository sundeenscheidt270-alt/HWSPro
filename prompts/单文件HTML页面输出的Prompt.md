# B端前端原型生成Prompt（基于PRD）

---

## 【使用场景】
基于PRD文档（尤其是附件C：UI交互说明）快速生成可交互的前端原型Demo，用于需求评审和用户演示。

---

## 【第一步：从PRD提取信息】

在生成前端页面前，请先从PRD中提取以下信息：

### 1. 页面基本信息（从PRD主档提取）
- **页面名称**: 从PRD的"页面清单"中获取
- **业务对象**: 从PRD的"数据对象"中获取核心业务对象名称
- **主要用户角色**: 从PRD的"角色清单"中获取

### 2. 数据字段（从附件A提取）
- **头字段**: 从"字段清单"中提取Header字段（如单号、日期、供应商等）
- **行字段**: 从"字段清单"中提取Line字段（如商品、数量、金额等）
- **枚举值**: 从"枚举与字典清单"中提取状态、类型等枚举（如：10:待审核, 20:已批准）

### 3. 功能与操作（从附件C提取）
- **查询条件**: 从"查询区域"提取筛选字段和默认值
- **列表字段**: 从"列表区域"提取需要展示的列
- **操作按钮**: 从"功能区域"提取页面级和行级操作
- **表单字段**: 从"详情/弹窗"提取新增/编辑表单字段

### 4. 状态与权限（从附件B提取）
- **状态流转**: 从"状态机"提取所有状态值
- **操作权限**: 从"状态-操作映射"提取各状态下可执行的操作

---

## 【第二步：生成Mock数据】

根据提取的字段信息，生成20条Mock数据，要求：
- 字段类型与PRD一致（varchar→字符串，decimal→数字，datetime→日期）
- 枚举值使用PRD定义的值（不能自己编造）
- 数据要有业务合理性（如：已关闭的单据不能再编辑）

---

## 【第三步：生成页面】

### 【你的角色】
你是资深前端工程师与交互实现专家。

### 【目标/交付物】
请生成一个可直接打开运行的单文件页面：File: index.html（只输出这一份完整代码）。

### 【技术约束】
* 仅 HTML + Tailwind CSS（使用 CDN：`<script src="https://cdn.tailwindcss.com"></script>`）+ 原生 JavaScript（写在同文件 `<script type="module">` 内）
* 禁用 React/Vue/框架/TS/构建工具语法；不引入任何第三方 JS 库
* 图标全部用"内联 SVG"；图片只用占位图（placehold.co 或 picsum.photos），不得引用来源不明外链
* 可访问性最低要求：弹窗/抽屉/下拉支持 Esc 关闭、点击外部关闭；打开时聚焦到第一个可交互元素；Tab 不穿透遮罩

### 【页面结构】（根据PRD调整）
* **顶部导航**: Logo、全局搜索、通知、用户头像下拉
* **左侧侧边栏**: 菜单（可折叠，状态写入 localStorage）
* **主内容区**:
  - 指标卡（≥3，根据PRD的关键指标设计）
  - 筛选区（根据PRD的"查询条件"生成搜索+状态下拉+日期）
  - 数据列表（根据PRD的"列表字段"生成table或card）
  - 分页
* **详情抽屉或弹窗**: 根据PRD的"详情区"生成
* **新增/编辑表单弹窗**: 根据PRD的"表单字段"生成，包含必填校验
* **Loading/Empty/Error 三种状态**: 提供演示开关可切换

### 【交互实现】（根据PRD调整）
1. 侧边栏折叠/展开（localStorage 记忆）
2. 用户菜单下拉（点击外部/ESC 关闭）
3. 列表前端过滤（根据PRD的筛选条件实现）
4. checkbox 全选/反选 + 批量操作按钮（根据PRD的"批量操作"）
5. 新增/编辑弹窗：打开/关闭、必填校验（根据PRD的"必填字段"）、保存后更新列表
6. 操作按钮的状态限制（根据PRD的"状态-操作映射"，如：已关闭状态不显示删除按钮）
7. Toast（成功/失败，自动消失）
8. 分页（前端切片）

### 【数据与实现方式】
* 内置 mock 数据 20 条左右，**字段严格按照PRD定义**
* 使用 state 对象 + render() 统一渲染；事件用事件委托；对用户输入做基本转义再渲染
* 每个关键模块（筛选、列表渲染、弹窗、toast、分页）写清楚注释

### 【输出格式】
* 第一行写：`File: index.html`
* 然后一次性给出完整 HTML 代码（可直接复制运行）

---

## 【使用示例】

### 示例1：有PRD时
```
@单文件HTML页面.md
@采购订单PRD.md

请根据PRD生成采购订单列表页面的前端原型
```

### 示例2：无PRD时（快速演示）
```
@单文件HTML页面.md

页面主题：采购订单管理
请生成前端原型，包含：
- 查询条件：单号、供应商、状态、日期范围
- 列表字段：单号、供应商、金额、状态、创建时间
- 状态枚举：10:待审核, 20:已批准, 30:已关闭
- 操作：新建、编辑、删除、审批、关闭
```
